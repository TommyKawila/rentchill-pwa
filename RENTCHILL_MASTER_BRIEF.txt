================================================================================
RENTCHILL_MASTER_BRIEF: THE SINGLE SOURCE OF TRUTH (คัมภีร์ตั้งต้นโครงการ)
================================================================================
วัตถุประสงค์ของไฟล์นี้: ใช้สำหรับคัดลอกและวางเป็นข้อความแรกเมื่อเริ่มบทสนทนาใหม่กับ AI
เพื่อกู้คืนบริบท โครงสร้าง สถาปัตยกรรม และวิสัยทัศน์ของ RentChill ทั้งหมด 100% โดยไม่มีการตกหล่น

[ปรัชญาหลักของโครงการ]
- "น้อยแต่มาก" (Less is More): ตัดฟีเจอร์และเมนูขยะออกให้หมด เอาเฉพาะที่ใช้จริงและตรงจุด
- "ความซับซ้อนคือศัตรู": ทุกระบบต้องเรียบง่าย ทำงานแบบอัตโนมัติ ไม่สร้างภาระให้ผู้ใช้และผู้พัฒนา
- "ยึดมั่นความจริงเชิงบวก": การสื่อสารตรงประเด็น จริงใจ แก้ไขปัญหาอย่างสันติ ไม่โฆษณาเกินจริง

================================================================================
ส่วนที่ 1: สถาปัตยกรรมและการเขียนระบบ (TECHNICAL ARCHITECTURE & CODING SYSTEM)
================================================================================
1. เทคโนโลยีหลัก (Tech Stack):
   - Frontend/Backend Framework: Next.js 14+ (App Router, Server Components)
   - Styling: Tailwind CSS (เน้นสไตล์ Minimalist, High Contrast สีเทา-ดำ-ขาว-เขียว)
   - Database & Authentication: Supabase (PostgreSQL)
   - Mobile Window: LINE LIFF (เปิดเว็บแอปใน LINE) และพัฒนาโครงสร้างแบบ Progressive Web App (PWA)

2. สถาปัตยกรรมโค้ด (Architecture Pattern):
   - แยกส่วนการทำงานอย่างเด็ดขาดตามหลัก Modular (Separation of Concerns)
   - แยก Logic หลังบ้าน (Service Layer & Custom Hooks) ออกจากหน้าตา UI (Presentational Component)
   - ใช้หลัก "Headless UI Pattern" เพื่อให้สมองกลคำนวณแยกต่างหาก พร้อมสลับเปลี่ยนผิวหน้ากาก (Skin) หรือย้ายแพลตฟอร์มได้โดยโค้ดหลักไม่พัง
   - ระบบโครงสร้างข้อมูลแบบ Multi-tenancy แยกสิทธิ์การเข้าถึงข้อมูลของแต่ละหอพักผ่าน `property_id` 

3. โครงสร้างโฟลเดอร์ใน Cursor (Workspace Directory):
   src/
   ├── app/                  # Route แยกส่วน Public Profile และ Private Tenant Board
   ├── services/             # ข้อมูลดิบและการเชื่อมต่อฐานข้อมูล (Types & Supabase CRUD)
   ├── hooks/                # สมองกลควบคุมระบบ (เช่น useInvoiceEngine, usePaymentEngine)
   └── components/           # หน้ากาก UI
       ├── frames/           # กรอบจำลองหน้าจอมือถือสไตล์ Minimal (MobileFrame.tsx)
       └── skins/            # สกินหน้าจอแสดงผลต่างๆ (เช่น MinimalInvoiceSkin.tsx)

================================================================================
ส่วนที่ 2: กลยุทธ์การตลาดและการสร้างดีมานด์ (MARKETING & STRATEGY)
================================================================================
1. ช่วงตั้งไข่ (Beta Launch - The Trojan Horse):
   - คัดเลือกและให้สิทธิ์เจ้าของหอพัก 10 ตึกแรกใช้งานระบบฟรี 100% เพื่อทำ Stress Test ระบบ
   - ใช้วิธีจับมือทำ (Concierge Onboarding) ช่วยย้ายข้อมูล คีย์ข้อมูลห้องพักให้ เพื่อลดแรงเสียดทานการเริ่มต้น

2. กลยุทธ์ดึงดูดจากฝั่งลูกบ้าน (Tenant-Pull Strategy & Trust Mark):
   - ป้ายอัจฉริยะหน้าตึก (Digital Receptionist): ติด QR Code หน้าตึกให้คนนอกสแกนดูห้องว่าง ราคา และบรรยากาศได้ทันทีโดยไม่ต้องโทรหาเจ้าของ
   - ทำให้โลโก้ RentChill เป็นสัญลักษณ์แห่งความโปร่งใส ลูกบ้านยุคใหม่เห็นสัญลักษณ์นี้จะมั่นใจว่า ค่าน้ำค่ายุติธรรม มีประวัติการจ่ายเงินที่ตรวจสอบได้จริง

================================================================================
ส่วนที่ 3: กลยุทธ์การโฆษณาและกรวยขาย (ADVERTISING & FUNNEL)
================================================================================
1. โครงสร้างกรวยขายที่ไร้แรงเสียดทาน (Zero-Friction Ad Funnel):
   - ด่านที่ 1: ยิง Facebook Ads เจาะกลุ่มเจ้าของหอพักวัย 40+ ชี้เป้าปัญหาที่น่าปวดหัว (สลิปปลอม, จดมิเตอร์ผิด)
   - ด่านที่ 2: กดจากแอดแล้ววิ่งเข้า LINE OA ทันที ส่ง Welcome Card ต้อนรับที่เป็นมิตร
   - ด่านที่ 3: ให้คลิกเปิด Interactive PWA Demo (หน้าแอปสแกนจ่ายเงินและหน้าแผงควบคุมจำลอง) เพื่อให้เห็นความง่ายใน 10 วินาที

2. ทิศทางชิ้นงานโฆษณา (Creative Direction):
   - ยึดหลัก "กระจกเงาสองด้าน" (The Magic Mirror) แสดงให้เห็นภาพฝั่งลูกบ้านใช้งานง่ายและสถานะหลังบ้านของเจ้าของอัปเดตทันทีแบบเรียลไทม์
   - มุ่งเน้นไปที่ค่าใช้จ่ายต่อการได้ลูกค้าจริง (Cost-Per-Acquisition: CPA) มากกว่ายอดไลก์ยอดแชร์

================================================================================
ส่วนที่ 4: การบริหารจัดการการเงินและต้นทุน (FINANCIAL MODELING)
================================================================================
1. โมเดลรายได้ (Pricing Tiers):
   - Tier S (ขนาดเล็ก): 399 บาท/เดือน
   - Tier M (ขนาดกลาง): 699 บาท/เดือน
   - Tier L (ขนาดใหญ่): 1,299 บาท/เดือน

2. เป้าหมายทางการเงิน (1M THB Milestone):
   - ตั้งเป้าสร้างรายได้ประจำรายเดือน (MRR) อยู่ที่ 1,000,000 บาท/เดือน
   - คำนวณตามสัดส่วนเฉลี่ย ต้องมีฐานลูกค้าหอพักที่ใช้งานจริงประมาณ 1,430 ตึกทั่วประเทศ

3. การควบคุมต้นทุนแฝง (API Costs Control):
   - ควบคุมค่าใช้จ่ายระบบตรวจสอบสลิปอัตโนมัติ (Slip Verification API) และค่าข้อความยิงเตือน (LINE Push Message) โดยการคำนวณรวมในงบ Server และใช้วิธีดึงผ่านเว็บหน้าต่าง LIFF เพื่อเลี่ยงการเสียโควต้าข้อความฟรีของ LINE OA

================================================================================
ส่วนที่ 5: การบริหารความเสี่ยงและทางหนีทีไล่ (RISK MANAGEMENT & FALLBACKS)
================================================================================
1. ข้อบังคับทางกฎหมาย (PDPA Consent Gate):
   - ฝังเงื่อนไขการยินยอมเก็บข้อมูลแบบสั้น กระชับ เข้าใจง่าย (Checkbox ยอมรับสิทธิ์) ไว้ตั้งแต่หน้าแรกที่ลูกบ้านป้อนรหัส Invite Code เพื่อเข้าตึก ป้องกันข้อพิพาทเรื่องข้อมูลเบอร์โทรศัพท์และประวัติการเงิน

2. แผนรับมือกรณีสิ้นยุค LINE (Platform Dependency Fallback):
   - ด้วยสถาปัตยกรรมแบบ Headless UI ระบบแกนหลักของ RentChill ยืนด้วยลำแข้งตัวเองได้ 100% บนเว็บเบราว์เซอร์ปกติ
   - หาก LINE ปิดตัวลง ระบบจะสลับไปเรียกใช้บริการส่ง SMS แจ้งลิงก์บิลแทน (`NotificationService`) และเปิดระบบ Web Push Notification บนหน้าจอโฮม (Add to Home Screen) ของผู้ใช้ทันที
   - ฐานข้อมูลจะเก็บช่องข้อมูล `phone_number` ควบคู่กับ `line_user_id` เสมอ เพื่อใช้เบอร์โทรศัพท์เป็นกุญแจสำรองในการยืนยันตัวตนผ่าน OTP

3. เครื่องมือทุ่นแรงการย้ายตึกใหญ่ (Excel Importer):
   - ป้องกันเจ้าของตึกขี้เกียจคีย์ข้อมูล โดยออกแบบฟีเจอร์อัปโหลดไฟล์ตารางสเปรดชีต Template `.xlsx` เพียงคลิกเดียวระบบจะแปลงข้อมูลสร้างตึก 100 ห้องเสร็จภายใน 1 วินาที
